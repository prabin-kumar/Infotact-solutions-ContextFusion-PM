 # Week 1: IoT Telemetry Ingestion and Signal Processing

During the first phase of the project, the AI4I Predictive Maintenance dataset was loaded and prepared for analysis using Python. The raw telemetry data was cleaned by removing unnecessary columns, renaming features for better readability, and encoding categorical values into numerical format. After preprocessing, rolling statistical features such as rolling mean, rolling standard deviation, and rolling maximum were generated over multiple operational windows to capture the machine's behavior over time. Additional derived features, including temperature difference and an estimated power proxy, were also created to enrich the dataset. The processed data was then saved for use in the next stages of the predictive maintenance pipeline.
# Week 2: Contextual Data Fusion and Feature Engineering

The second phase focused on enriching the machine telemetry with external contextual information. Simulated environmental factors such as ambient temperature, factory load, and humidity were generated and aligned with the machine sensor data to create a unified dataset. Rolling statistics were also calculated for the contextual signals to capture temporal trends. The enriched dataset was then used to train and evaluate a predictive maintenance model. An ablation study was performed by comparing a model trained using only IoT sensor features with another trained using both IoT and contextual features. The comparison measured the impact of contextual information on predictive performance and demonstrated the complete workflow of context-aware predictive maintenance.
